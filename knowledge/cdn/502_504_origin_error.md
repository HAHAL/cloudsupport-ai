# CDN 502/504 源站异常排查

## 适用场景

适用于用户通过 CDN 访问业务域名时出现 `502 Bad Gateway`、`504 Gateway Timeout`、网关错误、回源失败、访问偶发超时等问题。

## 常见现象

- 浏览器或客户端返回 502、504。
- CDN 日志中出现 `origin_error`、`upstream reset`、`connect timeout`、`read timeout`。
- 直连源站正常，但经过 CDN 访问异常。
- 部分地域或部分运营商异常，其他区域正常。
- 静态资源正常，动态接口或大文件下载异常。

## 可能原因

- 源站服务不可用、端口未监听、进程异常退出。
- 源站防火墙、安全组、WAF 或 IP 白名单拦截 CDN 回源 IP。
- CDN 回源协议、端口、Host、SNI 或证书配置错误。
- 源站响应时间超过 CDN 回源超时时间。
- 源站负载过高，连接数耗尽，或后端依赖数据库、缓存响应慢。
- 跨地域回源链路延迟高、丢包或连接不稳定。

## 排查步骤

1. 获取异常 URL、域名、时间范围、状态码、request_id、用户地域和运营商。
2. 使用 `curl -I` 或浏览器开发者工具确认 CDN 返回的状态码和响应头。
3. 直连源站测试相同 URL，比较直连源站和 CDN 访问结果。
4. 检查 CDN 访问日志中的 `status`、`origin_status`、`request_time`、`upstream_response_time`。
5. 检查源站 Nginx、应用服务、负载均衡和安全组日志。
6. 确认 CDN 回源配置：源站地址、端口、协议、Host Header、HTTPS SNI、证书链。
7. 如果是 504，重点分析源站接口耗时、慢 SQL、线程池、连接池和第三方依赖。
8. 如果是部分区域异常，按地域和运营商聚合日志，判断是否存在链路质量问题。

## 客户需要提供的信息

- 异常 URL 和加速域名。
- 异常发生时间，至少精确到分钟。
- 客户端 IP、地域、运营商。
- CDN 返回头、request_id 或 trace_id。
- 源站地址、回源协议和端口。
- 源站访问日志、错误日志和应用日志。
- 是否近期变更过源站、防火墙、证书、WAF 或 CDN 配置。

## 升级专家/研发的条件

- 多地域大面积 502/504，且源站确认正常。
- CDN 日志显示回源连接异常，但客户源站侧无对应请求记录。
- CDN 节点疑似异常或同节点大量用户受影响。
- 已确认配置正确，但 HTTPS 回源握手仍失败。
- 业务中断超过 SLA 或影响重要客户。
