# TLS Certificate Issue 排查

## 适用场景

适用于 HTTPS 访问失败、证书过期、证书链不完整、浏览器提示不安全、TLS 握手失败、CDN HTTPS 回源失败等问题。

## 常见现象

- 浏览器提示证书过期、证书不受信任或域名不匹配。
- `curl` 返回 `SSL certificate problem`、`handshake failure`。
- CDN 访问返回 502，日志显示 HTTPS 回源握手失败。
- 部分老旧客户端无法访问 HTTPS。
- 证书更新后仍看到旧证书。

## 可能原因

- 证书已过期或尚未生效。
- 证书绑定的域名与访问域名不匹配。
- 缺少中间证书，证书链不完整。
- 私钥与证书不匹配。
- SNI 未开启或回源 Host/SNI 配置错误。
- TLS 版本或加密套件不兼容。
- CDN、负载均衡或源站上仍部署旧证书。

## 排查步骤

1. 使用浏览器查看证书详情，确认有效期、颁发机构和绑定域名。
2. 使用 `openssl s_client -connect domain:443 -servername domain` 检查证书链。
3. 使用 `curl -v https://domain/path` 查看 TLS 握手过程。
4. 检查 CDN、负载均衡、源站三处证书是否一致。
5. 确认证书和私钥是否匹配。
6. 检查是否启用了 SNI，尤其是多域名共享 IP 场景。
7. 若老客户端失败，检查 TLS 1.0/1.1/1.2/1.3 和 cipher suite 兼容性。

## 客户需要提供的信息

- 异常 HTTPS 域名和 URL。
- 浏览器或客户端错误截图。
- `curl -v` 或 `openssl s_client` 输出。
- 当前证书文件、证书链和私钥部署位置说明。
- 是否使用 CDN、负载均衡、WAF 或 HTTPS 回源。
- 证书更新时间和生效范围。

## 升级专家/研发的条件

- 控制台显示证书已更新，但边缘节点仍返回旧证书。
- HTTPS 回源配置正确但 CDN 持续握手失败。
- 特定客户端兼容性问题无法通过 TLS 配置解释。
- 证书链在平台侧处理结果与标准工具验证不一致。
