# Video First Frame Slow 排查

## 适用场景

适用于点播或直播播放首帧慢、播放器长时间 loading、点击播放后等待时间长等问题。

## 常见现象

- 用户点击播放后 3 秒以上才出现画面。
- 播放器卡在 loading 或 buffering。
- 首帧慢但播放开始后比较流畅。
- 部分地域、部分网络或部分终端首帧慢。
- HLS 播放需要下载 m3u8 和多个 ts/fmp4 分片后才出画。

## 可能原因

- DNS 解析、TCP 建连、TLS 握手耗时高。
- CDN 缓存未命中，首个 m3u8 或首个分片回源慢。
- m3u8 层级多，首屏请求链路长。
- 首个视频分片过大或 GOP 过长。
- 播放器预加载策略不合理。
- 源站响应慢或跨地域回源。
- 码率过高，弱网环境下首个分片下载慢。

## 排查步骤

1. 获取播放 URL、播放协议、终端类型、播放器版本和用户网络。
2. 使用浏览器 Network 或播放器日志拆分 DNS、TCP、TLS、TTFB、下载耗时。
3. 检查 m3u8、key、ts/fmp4 分片的 CDN 命中情况。
4. 对比同一视频在不同地域、运营商和终端的首帧耗时。
5. 检查转码参数：GOP、分片时长、码率、首片大小。
6. 优化播放器预加载、起播码率和超时重试策略。
7. 对热点内容执行 CDN 预热，降低首个分片回源概率。

## 客户需要提供的信息

- 播放 URL、视频 ID、播放协议和清晰度。
- 首帧耗时数据和播放器日志。
- 用户地域、运营商、终端型号和 App/浏览器版本。
- m3u8 和首个分片请求的 Network HAR。
- 转码配置、GOP、分片时长和码率。
- 是否近期发布新播放器、切换 CDN 或调整转码模板。

## 升级专家/研发的条件

- CDN 命中正常但首帧仍显著高于基线。
- 多地域、多终端同时出现首帧慢。
- 播放器日志显示 SDK 内部状态异常。
- 转码或封装结果疑似不符合播放器预期。
