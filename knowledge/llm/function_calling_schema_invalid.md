# Function Calling Schema Invalid 排查

## 适用场景

适用于大模型 Function Calling、Tool Calling 或结构化输出中出现 schema 校验失败、参数缺失、类型错误、函数未触发等问题。

## 常见现象

- 模型返回的 JSON 无法通过 schema 校验。
- 必填字段缺失或字段类型不符合预期。
- 数组、枚举、嵌套对象格式错误。
- 模型直接回答文本，没有调用函数。
- 同一 schema 在复杂输入下失败率升高。

## 可能原因

- JSON schema 过于复杂，嵌套层级太深。
- 字段描述不清晰，模型难以理解业务含义。
- enum 值过多或命名相似。
- required 字段过多，输入信息不足。
- Prompt 没有明确要求优先调用工具。
- 服务端没有对工具参数做二次校验和修复重试。

## 排查步骤

1. 简化 schema，减少嵌套层级和可选复杂结构。
2. 为每个字段补充清晰 description，说明类型、单位和取值范围。
3. 将 enum 值控制在必要范围，避免相似命名。
4. 在 Prompt 中明确工具调用时机和缺失信息处理方式。
5. 对模型返回参数做服务端 JSON schema 校验。
6. 校验失败时，将错误信息反馈给模型进行一次修复重试。
7. 对高风险操作增加人工确认或白名单校验。

## 客户需要提供的信息

- Function/Tool schema 完整定义。
- 用户输入样例。
- 模型原始返回内容。
- 服务端校验错误信息。
- 使用的模型、SDK 版本和调用参数。
- 是否启用了 parallel tool calls 或 strict schema。

## 升级专家/研发的条件

- strict schema 下仍返回非法参数。
- 简化 schema 后仍稳定复现格式错误。
- 工具调用行为与官方文档不一致。
- 参数错误可能导致高风险业务操作。
