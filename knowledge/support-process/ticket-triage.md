# Ticket Triage

## Scenario

适用于企业技术支持团队对客户问题进行初步分类、优先级判断、负责人建议和首轮处理动作生成的场景。

## Triage Dimensions

- 问题类型：API、登录、权限、日志、部署、AI 应用、知识库问答。
- 严重程度：是否影响核心业务、是否大范围发生。
- 客户等级：免费、标准、企业或关键客户。
- 证据完整度：是否包含 request_id、时间、日志、复现步骤。

## First Response Actions

1. 确认问题现象和影响范围。
2. 收集 request_id、时间戳、账号、环境和复现步骤。
3. 根据问题类型给出首轮排查建议。
4. 判断是否需要升级研发、运维或平台团队。

## Required Information

- 工单标题和完整描述
- 客户等级和受影响产品
- request_id、错误码、日志片段
- 业务影响和发生时间

## Escalation Criteria

- 关键业务路径不可用。
- 多个用户或租户受影响。
- 一线排查无法确认根因，需要后端 trace 或平台侧信息。
